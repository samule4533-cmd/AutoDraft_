# chunker.py

## 개요
`chunker.py`는 PDF에서 변환된 Markdown 문서를 RAG(Retrieval-Augmented Generation)용 청크 단위로 분할하는 모듈이다.  
단순 길이 기준 분할이 아니라, **헤더 구조**, **문단 경계**, **표 형식**, **문장 단위 흐름**을 고려하여 검색과 응답 생성에 적합한 형태로 청크를 생성하는 것을 목표로 한다.

이 모듈의 핵심 목적은 다음과 같다.

- 문서 구조를 최대한 유지한 채 검색 가능한 단위로 분할
- 표와 일반 텍스트를 서로 다른 방식으로 처리
- 지나치게 긴 섹션을 안정적으로 분해
- 각 청크에 검색 및 후처리에 필요한 메타데이터를 부여

---

## 역할
이 파일은 Markdown 텍스트를 입력받아, 벡터 DB 적재에 적합한 `list[dict]` 형태의 청크 목록으로 변환한다.

특히 다음과 같은 책임을 가진다.

- Markdown 헤더(`#`, `##`, `###` 등)를 기준으로 1차 섹션 분리
- 긴 섹션을 문단 단위로 재분해
- 표 문단은 헤더 행을 보존하면서 행 그룹 단위로 분리
- 일반 텍스트 문단은 문장 경계와 줄바꿈을 기준으로 분리
- 각 청크에 `chunk_type`, `has_table`, `chunk_position` 등의 메타데이터를 추가

즉, 이 모듈은 **문서 구조 보존**과 **검색 효율성 확보** 사이의 균형을 맞추는 청킹 계층이다.

---

## 설계 의도
RAG 시스템에서 청킹 품질은 검색 정확도와 답변 품질에 직접적인 영향을 준다.  
이 파일은 단순 고정 길이 분할 방식의 한계를 줄이기 위해 다음 원칙으로 설계되었다.

### 1. 헤더 기반 구조 보존
문서를 무작위로 자르지 않고 Markdown 헤더 기준으로 먼저 나누어, 각 청크가 어느 섹션에 속하는지 명확히 유지한다.

### 2. 표와 일반 텍스트를 분리 처리
표는 일반 문단처럼 자르면 의미가 쉽게 깨지므로, 헤더 행과 구분선을 유지한 채 데이터 행 중심으로 나눈다.

### 3. 문장 경계 우선 분리
긴 텍스트 문단은 문장 단위로 먼저 분리하여 의미 단절을 줄이고, 필요 시 줄바꿈 단위로 추가 분할한다.

### 4. 헤더 전용 청크 제거
내용 없이 헤더만 존재하는 섹션은 검색 시 노이즈가 되기 때문에 청크 생성 대상에서 제외한다.

### 5. 후속 확장을 고려한 메타데이터 설계
`chunk_position`, `has_table`, `chunk_type` 등을 저장하여, 이후 계층적 청킹이나 검색 필터링 로직으로 확장할 수 있도록 했다.

## 주요 함수

### `_is_table_paragraph(text: str) -> bool`
문단이 표 형식인지 판별하는 내부 헬퍼 함수이다.

#### 동작 방식
- 빈 줄을 제외한 각 행을 검사
- 전체 행 중 `|`로 시작하는 행 비율이 50% 이상이면 표로 판단

#### 사용 목적
표 문단과 일반 텍스트 문단을 서로 다른 방식으로 처리하기 위한 1차 판별 단계이다.

---

### `_split_table_paragraph(text: str, max_len: int) -> List[str]`
마크다운 표를 청크 단위로 분리한다.

#### 동작 방식
- 표의 헤더 행과 구분선(`| --- |`)을 먼저 찾음
- 데이터 행을 순서대로 누적
- `max_len`을 초과하면 헤더를 유지한 상태로 새 청크 생성
- 각 청크는 독립적으로 읽어도 표 구조를 이해할 수 있도록 헤더를 반복 포함

#### 사용 목적
표 데이터가 여러 청크로 나뉘더라도, 각 청크가 단독으로 해석 가능하도록 만들기 위함이다.

#### 특징
- 표 헤더를 반복 포함하여 문맥 의존도를 낮춤
- 구분선이 없는 비정형 표는 원본 그대로 반환

---

### `_split_text_paragraph(text: str, max_len: int) -> List[str]`
긴 일반 텍스트 문단을 자연스럽게 분할한다.

#### 동작 방식
1. 한국어 문장 경계 우선 분리
2. 여전히 길면 줄바꿈 기준으로 추가 분리

#### 사용 목적
의미 단위가 중간에서 잘리는 문제를 줄이고, 검색 시 더 자연스러운 청크를 만들기 위함이다.

#### 특징
- 한국어 종결 표현과 문장부호를 기준으로 우선 분리
- 지나치게 긴 단락은 줄바꿈 단위로 강제 분할
- 단어 또는 어절 중간 절단을 최대한 피하려는 구조

---

### `_preprocess_paragraph(text: str, max_len: int) -> Tuple[List[str], bool]`
문단을 표 또는 일반 텍스트로 판별한 뒤 적절한 분할 함수를 호출한다.

#### 반환값
- 분리된 조각 리스트
- 표 여부(`is_table`)

#### 사용 목적
문단 처리 분기를 한 곳에서 통합하여 메인 로직을 단순하게 유지하기 위함이다.

---

### `split_markdown_into_chunks(...) -> List[Dict[str, Any]]`
이 모듈의 메인 함수이며, Markdown 전체 문서를 최종 청크 목록으로 변환한다.

#### 입력
- `markdown_text`: 청킹 대상 Markdown 본문
- `document_id`: 문서 식별자
- `source_pdf`: 원본 PDF 경로
- `model_name`: 문서 변환에 사용한 모델명
- `section_max_len`: 섹션 전체를 단일 청크로 유지할 최대 길이
- `group_max_len`: 문단 그룹 또는 표 조각의 최대 길이

#### 출력
벡터 DB 적재에 사용할 청크 리스트. 각 원소는 다음 구조를 가진다.

```python
{
    "chunk_id": "...",
    "header": "...",
    "text": "...",
    "metadata": {
        "document_id": "...",
        "source_file": "...",
        "section_order": ...,
        "header": "...",
        "source": "...",
        "model": "...",
        "mode": "...",
        "chunk_type": "...",
        "has_table": ...,
        "chunk_position": "only|first|middle|last"
    }
}

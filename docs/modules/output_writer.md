# output_writer.py

## 개요
`output_writer.py`는 파싱 결과를 파일 형태로 저장하는 책임을 담당하는 모듈이다.  
문서 파싱 이후 생성되는 Markdown, 전체 JSON, 필드 추출 결과, 파싱 리포트, 벡터 적재용 청크 파일 등을 일관된 형식으로 저장할 수 있도록 구성되어 있다.

이 파일의 핵심 목적은 다음과 같다.

- 파싱 결과의 저장 포맷을 표준화
- 파일 저장 로직을 한 곳에 모아 재사용성 확보
- 문서 전체 결과와 벡터 적재용 결과를 분리 저장
- 후속 단계(RAG, DB 적재, 디버깅)에 필요한 산출물을 일관되게 생성

즉, 이 모듈은 **파싱 결과를 실제 파일 시스템에 구조화된 형태로 기록하는 저장 전담 계층**이다.

---

## 역할
이 파일은 파싱 및 청킹 결과를 다양한 출력 포맷으로 저장하는 역할을 한다.

구체적으로 다음 책임을 가진다.

- 문서 전체 정보를 하나의 JSON 구조로 조합
- Markdown 원문 저장
- 전체 문서 JSON 저장
- 파싱 리포트 저장
- 필드 추출 결과 저장
- 벡터 DB 적재용 청크 리스트 저장

즉, 파싱/청킹/추출 자체를 수행하는 모듈은 아니며,  
그 결과를 **다음 단계에서 활용할 수 있는 저장물로 정리하는 후처리 유틸리티**이다.

---

## 이 파일이 필요한 이유
문서 파싱 파이프라인에서는 결과가 여러 형태로 생성된다.

예를 들어 다음과 같은 산출물이 필요할 수 있다.

- 사람이 읽을 수 있는 Markdown
- 전체 문서 구조를 포함한 JSON
- 필드 추출 결과만 따로 정리한 JSON
- 파싱 품질과 상태를 확인하기 위한 리포트
- 벡터 DB 적재용 청크 리스트

이런 저장 로직이 여러 파일에 흩어지면 다음 문제가 생긴다.

- 저장 포맷이 파일마다 달라질 수 있음
- 파일명 규칙이 일관되지 않음
- 저장 옵션 제어가 어려움
- 유지보수 시 수정 지점이 분산됨

`output_writer.py`는 이를 방지하기 위해 **출력 포맷 생성과 파일 저장 로직을 분리하여 관리하는 모듈**로 설계되었다.

## 주요 구성 요소

### `korean_ratio(s: str) -> float`
문자열에서 한글 비율을 계산하는 함수이다.

#### 동작 방식
- 문자열이 비어 있으면 `0.0` 반환
- 정규표현식으로 한글 글자 수를 계산
- 전체 길이 대비 비율 반환

#### 사용 목적
문서 통계 정보에 포함할 `korean_ratio` 값을 계산하기 위함이다.

#### 의미
문서가 한국어 중심인지, OCR 결과나 파싱 결과가 예상과 크게 어긋나지 않았는지 확인하는 간단한 품질 지표로 활용할 수 있다.

---

### `build_document_json(...) -> Dict[str, Any]`
파싱 결과 전체를 하나의 문서 JSON 구조로 조합하는 함수이다.

#### 입력
- `source_pdf`: 원본 PDF 경로
- `markdown_text`: 파싱된 Markdown 본문
- `chunks`: 청킹 결과 리스트
- `elapsed_sec`: 파싱 소요 시간
- `fields`: 추출된 필드 정보
- `image_count`: 처리된 이미지 수
- `model_name`: 사용한 모델명

#### 반환값
문서 전체를 설명하는 JSON 딕셔너리

#### 포함 정보
- 원본 파일 경로
- 문서 식별자
- 문서 포맷
- 파싱 모드
- 사용 모델
- 소요 시간
- Markdown 본문
- 추출 필드
- 청크 리스트
- 통계 정보

#### `stats` 필드
`stats`에는 다음과 같은 요약 정보가 포함된다.

- `text_len`: 전체 텍스트 길이
- `chunk_count`: 생성된 청크 수
- `korean_ratio`: 한글 비율
- `image_count`: 처리된 이미지 수

#### 사용 목적
전체 파싱 결과를 하나의 구조화된 JSON으로 저장하거나, 이후 DB 저장/분석/디버깅에 재사용하기 위함이다.

---

### `save_outputs(...)`
여러 종류의 결과 파일을 실제 디렉터리에 저장하는 함수이다.

#### 역할
저장 옵션에 따라 필요한 산출물만 선택적으로 파일로 기록한다.

#### 입력
- `output_dir`: 저장 대상 디렉터리
- `source_pdf`: 원본 PDF 경로
- `markdown_text`: Markdown 본문
- `document_json`: 전체 문서 JSON
- `parse_report`: 파싱 리포트
- `fields`: 필드 추출 결과
- `chunks`: 벡터 적재용 청크 리스트

#### 저장 옵션
- `save_final_md`
- `save_final_json`
- `save_parse_report`
- `save_fields_json`
- `save_vector_chunks`

각 옵션은 `True`일 때만 해당 파일을 저장한다.

## 수행 작업

### 1. `src/rag_chain.py` — Phase 1 신규 작성 (약 450줄)

RAG 파이프라인의 핵심 오케스트레이션 레이어를 작성.

**구현 내용**
- `RAG_CONFIG`: threshold, n_results, max_context_chars 등 튜닝 포인트 단일 집중
- `retrieve()`: ChromaDB 검색 결과 평탄화 (중첩 리스트 → list[dict])
- `filter_by_threshold()`: cosine distance 기준 노이즈 청크 제거
- `build_context_block()`: 통과 청크를 LLM 컨텍스트 문자열로 조합, max_chars 초과 시 하위 rank 청크 제외
- `build_prompt()`: 시스템 지시 + (이전 대화) + 참고 문서 + 질문 조합
- `generate_answer()`: Gemini 호출, 429 rate limit 지수 백오프 재시도, RPD 초과 즉시 중단
- `format_citations()`: 출처 중복 제거
- `ask()`: 전체 플로우 오케스트레이션, 단계별 fallback 분리
- `RagResult` dataclass: 검색 통계(retrieved_count, passed_threshold, top_distance) + fallback 사유 포함
- CLI `--test` 모드: 8개 테스트 케이스(있는 질문 / 없는 질문 / 수치 포함 / 엉뚱한 질문) 배치 실행

**설계 원칙**
- ChromaDB / Gemini를 직접 건드리지 않고 `vector_db` / `llm_api` 함수만 호출하는 순수 오케스트레이션 레이어
- threshold 통과 청크 0개이면 LLM 호출 자체를 생략 → 환각 방지 강제
- 검색 오류 / LLM 오류 / 문서 없음을 각각 다른 `fallback_reason`으로 분리 → 원인별 디버깅 가능

### 2. `src/rag_chain.py` — Phase 2 대비 고도화 (두 번째 커밋)

**`build_context_block` 헤더 정제**
- `chunk['header']`에 `## 출원일자` 처럼 `#` 기호가 그대로 있어 LLM 인용 시 `[출처: ## 출원일자]`처럼 노출되는 문제 수정
- `re.sub(r"^#{1,6}\s*", "", header)` 적용으로 LLM에 전달하는 컨텍스트에서 `#` 제거

**`generate_answer` 빈 응답 처리**
- Gemini 안전 필터 차단 또는 모델 오작동 시 빈 문자열 반환 케이스 추가 방어
- 빈 응답을 `GeminiAPIError`로 raise → `ask()`의 llm_error fallback으로 흡수

**`format_citations` 중복 기준 변경**
- 기존: `chunk_id` 기준 dedup → 동일 섹션 청크가 여러 개 통과하면 같은 출처가 중복 표시되는 문제
- 변경: `(header, source_file)` 조합 기준 dedup → 같은 섹션은 출처 하나로 묶음
- `build_context_block`과 동일하게 `#` 제거 처리 → LLM 인라인 인용과 Citation 목록 헤더 표기 일치
